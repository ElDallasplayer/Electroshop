Sanasto
802.11b tai 802.11g
802.11b -verkot voivat toimia enintään nopeudella 11 Mb/s. 802.11g-verkkojen nopeus on enintään 54 Mb/s.

Tukiasema
Kutsutaan myös langattomaksi reitittimeksi. Tukiasema on langattoman verkon keskus ja ohjaa sen turvamäärityksiä. Jos verkossa on tukiasema, kyseessä on infrastruktuurityyppinen verkko.

Ad-hoc
Jokainen laite kommunikoi suoraan kunkin erillisen laitteen kanssa. Tukiasema ei ohjaa langatonta tietoliikennettä. Tämän tyyppistä verkkoa kutsutaan myös langattomaksi vertaisverkoksi.

APIPA
Automatic Private IP Addressing (APIPA). Tätä käytetään, kun verkkolaite ei pysty hankkimaan IP-osoitetta DHCP-palvelimelta.

ASCII
American Standard Code for Information Interchange (ASCII). Merkin määritys numeerisessa muodossa. Esimerkiksi A-kirjaimen ASCII-koodi on 65.

Todentamismenetelmä
Verkossa käytetyn langattoman todentamisen tyyppi.

Kanava.
Langattomat verkot käyttävät kanavia. Kukin kanava käyttää eri radiotaajuutta ennakkoon määritetyllä taajuusalueella. Kaikki langattoman verkon langattomat laitteet käyttävät samaa kanavaa, ellei niitä ole kytketty toisiin tukiasemiin, jotka toimivat yhdyskäytävänä langattomaan verkkoon.

DHCP
Dynamic Host Configuration Protocol (DHCP) -palvelin jakaa IP-osoitteet verkkolaitteille.

DSL
Digital Subscriber Line (DSL), josta myös käytetään lyhenteet ADSL ja xDSL. Nopea Internet-yhteys.

EAP
Extensible Authentication Protocol (EAP) toimii langattomien ja kiinteiden verkkojen todentamisen kehyksenä. EAP käyttää RADIUS-palvelinta, joka todentaa yritysverkkojen käyttäjät ennen kuin heidät päästetään langattomaan tai kiinteään verkkoon.

Verkkokaapeli (Ethernet-kaapeli).
Kaapeli, jolla laite kytketään langattomaan verkkoon. Ethernet-kaapeleissa käytetään RJ45-liittimiä, ja kaapelia itseään kutsutaan mm. suorakaapeliksi.

Yhdyskäytävän osoite.
Yhdyskäytävän osoitteen avulla laite voi kommunikoida toisissa verkoissa olevien laitteiden kanssa. Yhdyskäytävän osoitetta ei tarvitse yleensä määrittää.

HEKS.
HEKSADESIMAALIN lyhenne. Lukujärjestelmä, jossa käytetään numeroita 0–9 ja kirjaimia A–F.

Infrastruktuuri
Kukin laite kommunikoi yhteisen tukiaseman kautta. Tämä on yleisin langattoman verkon tyyppi.

IP-osoite.
Jokaisella verkossa olevalla laitteella on oltava ainutlaatuinen osoite. Tätä osoitetta kutsutaan IP-osoitteeksi. Se annetaan manuaalisesti tai se voidaan määrittää automaattisesti DHCP-palvelimen avulla. IP-osoite on pakollinen TCP/IP-verkoissa.

LEAP
Lightweight Extensible Authentication Protocol (LEAP). Ciscon protokolla, jota käytetään langattomien lähiverkkojen 802.1X-todentamiseen. LEAP käyttää EAPin tavoin RADIUS-palvelinta, joka todentaa yritysverkkojen käyttäjät ennen kuin heidät päästetään langattomaan tai kiinteään verkkoon.

MAC-osoite
Media Access Control (MAC) -osoite on ainutlaatuinen 12-numeroinen luku, joka määritetään jokaiselle verkkolaitteelle.

Solmun nimi
Langattoman laitteen uniikki nimi. Solmun nimi perustuu yleensä MAC-osoitteeseen.

Signaalin voimakkuus
Laitteen ja sen käyttämän tukiaseman välisen langattoman signaalin voimakkuus.

Aliverkon peite.
IP-osoitteeseen tarvitaan myös aliverkon peite. Aliverkon peite voidaan antaa manuaalisesti tai määrittää automaattisesti DHCP-palvelimen avulla. Aliverkon peite on pakollinen TCP/IP-verkoissa.

SSID
Service Set Identifier (SSID) on verkon nimi, jota langaton verkkosi käyttää. Joissain langattomissa verkoissa SSID-nimeä ei ilmoiteta turvallisuuden parantamiseksi.

Kytkin
Kytkimen avulla useat verkkolaitteet voivat kommunikoida keskenään.

TKIP
TKIP (Temporal Key Integrity Protocol) poistaa ennustettavuuden, jonka avulla tunkeutujat pystyivät käyttämään hyväksi manuaalisesti määritettävien WEP-avainten turva-aukkoja. TKIP-avaimet luodaan automaattisesti ja ne ovat pitempiä kuin WEP-avaimet. Näin ne ovat turvallisempia kuin tavalliset WEP-avaimet.

WEP-avain
Wireless Equivalent Protocol (WEP) -avain on langattoman verkon perustason turvaominaisuus, joka suojaa verkkoa satunnaisilta salakuuntelijoilta. WEP-avainta kutsutaan myös verkkoavaimeksi.
